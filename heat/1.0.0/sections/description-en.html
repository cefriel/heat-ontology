Hydrogen Valley Ontology (H2V): Description back to ToC

The entry point of the ontology is the class [Tool](https://w3id.org/heat#Tool), defined as a structured conceptual system that explains phenomena by specifying key constructs and the relationships between them. Tools are associated with a [Context](https://w3id.org/heat#Context), that is the specific environment, application domain, or scenario in which a tool or a construct is applied, providing the situational framework that influences how concepts are interpreted, measured, and utilized. They are linked to resources (like papers) presenting them with the property [isReferencedBy](http://purl.org/dc/terms/isReferencedBy). The property [hasModelDiagram](https://w3id.org/heat#hasModelDiagram) represents the reference to the tool's graphical visualisation, while [isOriginal](https://w3id.org/heat#isOriginal) indicates whether the tool is validated as a standard, i.e. whose constructs and questions have not been defined based on another reference tool, or not. A Tool may also be linked to a [Reference Theory](https://w3id.org/heat#ReferenceTheory), which is a standardized, conceptual theory that aggregates related tools, models, or variants under a unified theoretical structure. It serves as the canonical reference point to which all specific implementations or adaptations (tools/models) are linked. Each tool is defined by a set of [Constructs](https://w3id.org/heat#Construct), that are concepts representing specific attributes in a given context. A Construct may be linked to a [General Construct](https://w3id.org/heat#GeneralConstruct), which is a generalised concept that serves as a reference point for aggregating all related constructs from different tools, enabling the systematic organisation and comparison of equivalent constructs in different contexts. Constructs are associated with a series of measurement elements (i.e., [Questions](https://w3id.org/heat#Question)) that allow for an objective evaluation of the phenomenon analysed by the tool. Each question is linked to its [questionText](https://w3id.org/survey-ontology#questionText), and its [Likert Answer](https://w3id.org/heat#LikertAnswer), which is a subjective response format in which a user selects a point on an ordered categorical scale to indicate the intensity or extent of his/her perception, evaluation, or experience regarding a given item. A Likert Answer is associated with an ordered numerical range in which it is defined ([numericalValueRange](https://w3id.org/heat#numericalValueRange)) and an ordered textual range ([textualValueRange](https://w3id.org/heat#textualValueRange)). Moreover, the property [isScaleReversed](https://w3id.org/heat#isScaleReversed) indicates whether the scale of the Likert Answer is reversed compared to the meaning of the construct it is associated with, or not. All classes of the ontology can be associated with a name (property [title](http://purl.org/dc/terms/title)), and a [description](http://purl.org/dc/terms/description).
